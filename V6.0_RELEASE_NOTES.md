**Release Date**: December 28, 2025  
**Codename**: "Dream"

Version 6.0 introduces the **Dream Layout**, a revolutionary Map-First architecture that transforms the driver experience from a cluttered dashboard to an intelligent co-pilot interface.

#### Key Features

**1. Map-First Architecture**
- Full-screen map is now the foundation of the UI
- All interface elements float transparently above the map
- Never lose sight of your surroundings while accessing data

**2. Micro HUD Header**
- Minimalist floating header with only essential information:
  - Live connectivity status
  - Current earnings at a glance
  - GPS indicator
- Removes clutter while keeping you informed

**3. Interactive Bottom Sheet**
- **Collapsed State (Drive Mode)**: Shows only the top recommendation
- **Expanded State (Plan Mode)**: Reveals full dashboard with all widgets
- Drag to open/close or use buttons
- Smooth animations powered by Framer Motion

**4. Focus Mode**
- One-tap button to enter "driving focus"
- Hides all UI except the map
- Perfect for when you need zero distractions
- Toggle on/off instantly

**5. Glassmorphism 2.0**
- Transparent, blurred backgrounds
- Minimal borders and shadows
- Information hierarchy through typography, not boxes
- Dark mode optimized for night driving

## Technical Improvements

### New Components
- `DreamLayout.tsx` - Complete new layout system
- Bottom Sheet with drag interactions
- Micro HUD header component

### Enhanced Components
- `DraggableCardGrid` now supports "minimal" variant
- Transparent card styling option
- Reduced visual weight for better map visibility

### Architecture
- Map rendered at `z-0` (background layer)
- UI elements float at `z-10+` (foreground layers)
- State management for Focus Mode and Drawer states
- Smooth transitions between collapsed/expanded states

## Migration Guide

### For Users
- Dream Layout is now the **default layout**
- Access via theme selector or automatically on app load
- Previous layouts (Neon Cockpit, Pro Dashboard, etc.) still available
- Your preferences and data are preserved

### For Developers
- New `variant` prop on `DraggableCardGrid`: `'standard' | 'minimal'`
- `DreamLayout` exported from `features/layout`
- `AppLayout` defaults to `DreamLayout`
- All existing layouts remain functional

## Design Philosophy

### Before (v5.x)
- Dashboard-first: Show everything at once
- Grid of opaque cards covering the map
- High information density = high cognitive load
- Optimized for stationary planning

### After (v6.0)
- Map-first: Context is king
- Floating, transparent UI elements
- Contextual information based on mode
- Optimized for both driving and planning
